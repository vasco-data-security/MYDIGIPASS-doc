    A JWS is a string made of 3 parts separated by . as defined in rfc7515

    • JOSE Header
    • JWS Payload
    • JWS Signature

    JOSE header

    {
"typ": "JWT",
"alg": "RS256",
"kid": "5e02e631-77f3-4c7c-b36a-5a1ec13a48fb"
}

    JSON object containing the parameters describing the cryptographic operations and parameters employed.

    Key Description
    typ Type of token
    alg Algorithm used to sign the JWT
    kid Key id, the id of the public key used to sign the JWT the key itself is defined in the endpoint

    JWS Payload

    The JSON object containing the requested data.

    {
"iss": "https://mydigipass.com",
"sub": "the_user_account_uuid",
"aud": "your_applications_client_id",
"iat": 1499756471,
"exp": 1499756771
}

    This endpoint returns a set of JSON Web Keys that can be used to verify the signature of any JWS issued by MYDIGIPASS.

    Key Description
    kid Id of the JWK, will be referenced in the JOSE header of any JWS document delivered by MYDIGIPASS
    n Modulus of the JWK
    e Exponent of the JWK
    use Used for signature only

    Note that these headers are set in the response and should also be set on your own JWKS endpoint:

    Header Value
    Content-Type application/jwk-set+json
    Cache-Control The amount of time (in seconds) you can cache the response e.g max-age=86400, public

    Authorization Endpoint

    GET or POST /oauth/authenticate

    This is most commonly done by adding opendid to the data-scope attribute of the MYDIGIPASS button.

    The openid scope can be passed to our authorization endpoint.

    The openid LoA parameter can also be specified in this request, passing the requested value (one of “http://eidas.europa.eu/LoA/high”, “http://eidas.europa.eu/LoA/substantial” or “http://eidas.europa.eu/LoA/low”) to the key “acr_values”. Requiring “http://eidas.europa.eu/LoA/high” will prevent users from authenticating with an SMS code.

    Now, the token endpoint will also return an id_token param containing a JWS with the following payload.

    client_assertion JWS payload

    {
  "sub": "client_id",
  "aud": "http://mdp.test:3000/oauth/token",
  "exp": 1502185869,
  "jti": "0f6c227b-8596-46e0-a83e-9303b24fecdd"
}
    Claim Description
    sub Your client_id
    aud https://www.mydigipass.com/oauth/token
    exp The expiration time on or after which the client_assertion will be rejected expressed as a NumericDate
    jti A unique identifier, generated by you, that will invalidate any reuse of this client_assertion

    id_token JWS payload

    {
  "iss": "https://mydigipass.com",
  "sub": "the_user_account_uuid",
  "aud": "your_applications_client_id",
  "iat": 1499756471,
  "exp": 1499756771,
  "acr": "http://eidas.europa.eu/LoA/high"
}
    Claim Description
    iat The time at which the id_token was issued
    exp The expiration time on or after which the id_token will be rejected expressed as a NumericDate
    acr The Level of Assurance up to which the user is currently authenticated. Using the SMS authentication method or not having an eID profile will currently result in a less than “http://eidas.europa.eu/LoA/high” LoA.
